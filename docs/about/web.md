## Web 设计最佳实践

### 1. 用户是浏览者

访问者只会花几秒钟决定是否继续阅读，因此需要：  

- 在页面第一句突出核心观点
- 使用简短段落和有趣标题
- 提高内容的可扫描性

### 2. 少即是多

- 保持章节简短
- 避免冗长文字，影响用户体验
- 将大量内容分解成小模块，放在不同页面

### 3. 导航结构

- 在全站保持一致的导航结构
- 避免在文本段落内使用超链接
- 把链接放在段落底部或菜单中

### 4. 加载速度

- 大多数用户只会等待 7 秒以内
- 在低速连接测试网页
- 必要时删除重型图片或多媒体内容

### 5. 用户反馈

- 建立良好的反馈机制
- 访问者是你的"客户"
- 积极采纳来自不同领域的改善建议

### 6. 显示器兼容性

- 在不同分辨率的显示器上测试
- 避免仅针对高分辨率设计
- 确保低分辨率（如 800×600）也能正常显示

### 7. 浏览器兼容性

- 在多个浏览器测试网站
- 主流浏览器：IE、Firefox、Chrome
- 使用正确的 HTML 编码帮助浏览器正确渲染

### 8. 插件支持

- 音频、视频等多媒体可能需要插件
- 确保访问者拥有所需的播放软件
- 提供备选方案或说明
